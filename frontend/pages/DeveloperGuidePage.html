Architecture

IDMS is a Flask application with SQLite persistence and server-rendered Jinja pages.

  • src/main.py: routes, auth decorators, request parsing, file responses, CSV exports.
  • src/database.py: schema, migrations, validation, persistence, reports.
  • frontend/pages: HTML templates.
  • src/test_*.py: unittest regression suites using temporary databases.

Database Schema

users_data -> votes, meeting_attendance, uploads document_categories -> uploads auth_email_links -> registration/password reset import_jobs -> import_rows -> import_changes activity_log -> activity summaries voting_events -> voting_options -> votes meetings -> meeting_attendance, meeting_minutes financial_transactions -> financial reports budgets -> budget_categories, budget_alerts notifications -> delivery/history log bug_reports -> defect workflow

Primary APIs

  • Voting: `/api/voting/events`, `/api/voting/votes`, results JSON/CSV.
  • Documents: `/api/documents`, file download routes.
  • Meetings: `/api/meetings`, attendance JSON/CSV, minutes APIs.
  • Finance: transactions, budgets, budget alerts, report JSON/CSV.
  • Admin: members, imports, activity summary.
  • Notifications: list and process due notifications.

Security

  • `login_required` protects authenticated pages and APIs.
  • `admin_required` protects admin workflows.
  • Member passwords are bcrypt hashed.
  • Email verification and reset links are signed tokens.
  • Downloads use secure filenames, private cache controls, and audit logging.

Maintenance Checklist

  1. Add schema and migrations in `database.init_db()`.
  2. Validate inputs in `src/database.py` helper functions.
  3. Keep Flask routes thin and protected by the correct decorators.
  4. Log activity for admin-visible workflows.
  5. Add or update focused unittest coverage.
  6. Update user and technical documentation.
  7. Run `python -m unittest discover -s src` before publishing.
